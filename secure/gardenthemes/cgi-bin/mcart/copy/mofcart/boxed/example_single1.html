HREF and GET input Return to Examples

Use: You want to submit an item to the cart via a hyper link or GET

Note: The packaged installation has this input mode disabled by default. You must turn it on in the Front End configuration file to enable it. See documentation about Front End Configurations.

Security Warning: These methods parse product information via the QueryString and the product's data can easily be altered by changing values in the browser's address bar and URL string. You should not use these methods of linking products to the cart if you are at all concerned about price tampering.

Instructions: Click on the link, image, or button associated with a product. It will be added to the cart.

Product: World Class BaseBall Cards, Set of 14. Click Here to Order Now

Product: World Class BaseBall Cards, Set of 10.

Product: World Class BaseBall Cards, Set of 100

FORM syntax ..  

<a href="http://www.gardenthemes.com/cgi-bin/mcart/mof.cgi?order=14----BaseBall+Cards----World+Class+BaseBall+Cards----0.35----1----1">Click Here to Order Now</a>

<a href="http://www.gardenthemes.com/cgi-bin/mcart/mof.cgi?order=10----BaseBall+Cards----World+Class+BaseBall+Cards----0.55----1----1"><img src="buttbuy.gif" width=37 height=17 border=0 alt="Click to Add to Cart" align="absbottom"></a>

<FORM method=GET action="http://www.gardenthemes.com/cgi-bin/mcart/mof.cgi">
<input type="hidden" name="order"
value="100----BaseBall Cards----World Class BaseBall Cards----0.08----1----1">

<input name="update_btn" type=submit value="Add To Cart">
</FORM>

Notes

This is the simplest way of getting items into your cart; however, you should consider using any of the other POST methods. These methods leave openness for tampering with the products information, pricing, etc. See more about this on our support pages ..

Note that quantity is submitted as the first field placeholder in the order input
value="Qty----Item----Desc----Price----ShipCode----TaxCode"

These methods don't require any postmode
You must construct strings with the " + " seperator for blank spaces
Don't use any of these characters: ` ! @ # $ % ^ & * ( ) { } / \ < >
Don't use any Perl special characters, quotes, semicolon, etc.
Don't use the | character
Don't use the double colon ::
Don't use the square bracket characters [ or ]
