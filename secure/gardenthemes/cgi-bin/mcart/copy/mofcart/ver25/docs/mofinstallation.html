 How do I get the orders ?

Each time an order is completed on your site, the cart will send an email notice to the Merchant. The cart also sends an email summary to the customer. New order notices are sent to:

  • orders@gardenthemes.com

The mail notice does not include sensitive information like credit card numbers. To obtain a credit card number you must log into the CC retrieval utility, and obtain the card numbers over a secure connection. You will need the login & password (listed in the settings.html reference you downloaded & saved on your local computer).

The CC retrieval utility is located here:

  • https://secure32.elinuxservers.com/~gardenth/cgi-bin/mcart/admin/mofcc.cgi

How do I make the cart look like my site ?

MerchantOrderForm cart uses 13 templates that show up at various points in the ordering process. The 13 templates are standard HTML pages, and can be designed to match your site layout. You will need to be familiar with basic web page editing concepts to create your templates.

Consult the main documentation section Formatting Templates for detailed information on making your own templates.

The packaged templates were placed in the /mofcart/ directory on your site. Use the same location and filenames for your templates, and just replace the ones that came with the installation. You must follow the rules below when making your templates.

Rules for making templates:

  • The Input Marker below must be placed in your template. The marker must appear as an HTML comment line at the place in the template where you want the cart to insert its dynamic output. It must be on a line by itself. Double check that your html editor has kept it on a line by itself. The marker must be exactly like this

    <!--MOF-INPUT-AREA-->
  • The Window Naming JavaScript MAIN is also needed in each of the (non PopUp) Templates

    <script type="text/javascript" language="JavaScript">
    <!-- name and focus window
    window.name = 'MAIN';
    window.focus();
    // End -->
    </SCRIPT>
  • Always use a Full URL (http(s)://www.something.com/image.gif) to address all objects within your template page. The templates are processed through the CGI environment on your server, which do not consistently handle relative addressing correctly. If the cart is running via a secure (SSL) environment, then all objects in your templates must also be referenced using the secure (SSL) address (HTTPS), i.e., mofstyle.css, images.gif, other-objects.etc. If you are getting browser SSL "items insecure" messages when running the templates via SSL, then you have some object that is being called via standard HTTP protocol. Remember, even the nested objects must be addressed via HTTPS (SSL). For example, the mofstyle.css has URL image settitngs that must also be SSL addresses.

  • Always include the mofstyle.css CSS (style sheet) file in your templates. The cart's output is formatted entirely with CSS, which loads as part of your template. You can add more styles to the mofstyle.css, but do not delete any of the existing styles, or the dynamic output will not have the needed styles for formatting.

    IF you already have a complicated CSS style for your site layoug, then you can just add the "boxed" <mofstyle.css> that is smaller, containing just the CSS for dynamic output. You can find that in the /copy/mofcart/boxed/mofstyle.css, which you saved as backup.
  • The templates can be any HTML design.  Avoid anything too fancy, like a reloading Java menu.  When operating out of a server's CGI only your browser should be allowed to reload a script, so that it can rePOST the necessary data.

How do I change settings for the cart ?

MerchantOrderForm cart does not have an online admin utility. You must make all setting changes in the configuration files. If you are not familiar with editing Perl configuration files, then you should start with the documentation section About Configurations. You can find detailed explanations of all the cart settings in the documentation sections: Configurations Front End & Configurations Checkout.


 Where are the configuration files ?

There are 3 configuration files which direct the cart's behavior. We recommend that you backup the original installation configuration files and set them aside in a safe place. It is also a good idea to make backups of the configuration files as you make changes. Always make sure you upload the configuration files back to the same location on your server.

  • FileName : common.conf
    Keeps common path settings
    /home/gardenth/public_html/cgi-bin/mcart/common.conf

  • FileName : mof.conf
    Keeps all settings for the cart Front End, products, shipping, tax, etc.
    /home/gardenth/public_html/cgi-bin/mcart/lib/mof.conf

  • FileName : mofpay.conf
    Keeps all settings for the cart Back End, billing, checkout, etc.
    /home/gardenth/public_html/cgi-bin/mcart/lib/mofpay.conf

Saving Order Data in a mySQL database

This package of MOFcart (v2.5 10-21-03) has only the datasave utilities, and will be of little use to you unless you are developing your own mySQL interface to do something with the MOFcart order data once it is saved. You can use any of the commercially available mySQL tools for mySQL dB management to build queries, etc.

Lots more mySQL stuff is planned for evolving versions. Hopefully, you will just need to "drop in" the evolving mySQL scripts as they are developed. Check the updates area for anything new along the mySQL lines.

Follow the steps below to enable mySQL datasave from MOFcart orders. We recommend that you not use a remote dB, since the data transfer is only secure within the same server as the MOFcart scripts.

What about ARES (Affiliate Referral Earnings System)?

The ARES v2.5 affiliate earnings system is installed with MOFcart v2.5.

http://www.gardenthemes.com/ares/

It is an entirely ascii flat file based system. Future versions of ARES & MOFcart v3.0 will evolve to complete mySQL based systems. Check the updates area for anything new along the mySQL lines.

What about security ?

Always host with a reputable company. Most commercial servers have good security. It is in their best interest to do so. We recommend you read the section Security Issues in the documentation. Security is your responsibility! Inexperienced CGI installers often leave common, yet fixable, security flaws when installing an application of this type. Do not be one of them.

One of the important security measures is to make sure that your configuration files are not accessible to unauthorized users, either by vieweing them from the web, or from traveling on the server.

From your web browser, click on the links below . If you can view the configuration files from your browser, then you have bad security. Consult Documentation > Security Issues.
