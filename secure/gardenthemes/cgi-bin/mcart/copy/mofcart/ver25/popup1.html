Notes:

  • Name and Focus all windows in the Pop Up scheme

  • INPUT options: All of the normal MOFcart POST rules can be used: SINGLEPOST, CHECKBOXES, QUANTITYBOXES. Cannot use the GET with POPUP mode

  • INPUT must include hidden flag for Pop Up status (case sensitive) So that MOFcart can tell your are POSTing in Pop Up mode else you will get incorrect error and/or validation messages. Do not send this field from non Pop Up pages, or else make sure it is Null value=""

    <input type="hidden" name="POP_TARGET" value="POPUP">
    Identifies the cart output is coming back to the Pop Up window

    May also target another window value="MAIN" and successful cart output will be sent there using MOFcart regular ViewCart template output, or custom message if using REDIRECT_URL

    All MOFcart Technical Error Messages and Validation Messages always return to the Pop Up window. Only successful cart Posts can change window placement.

  • INPUT options (Field Names) Note: Names are case sensitive
    • REDIRECT_URL value="Full URL"
      For "add to cart" successful outcome (optional). After successful POST, redirection to this URL. May include (#nameAnchor)(?vars=&GET=string). Note: be careful not to put a 404 destination here. if REDIRECT_URL not present, then MOFcart will use the Pop Up ViewCart default format

    • POP_TARGET value="POPUP" (mandatory).
      Make sure the value = POPUP to get results back to the Pop Up window. You can set it to another name (example: MAIN) and MOFcart will use that window (using a JavaScript redirect from Pop Up window), but be careful not to name a window that is not open, or most browsers will trigger a new window, which will confuse shoppers.

      Will work in conjunction with REDIRECT_URL.
      Example (target Pop Up, but use a static message or script):
      REDIRECT_URL = http://www.somesite.com/mofcart/page-or-script.*
      POP_TARGET = POPUP

      Also you can trigger the traditional ViewCart in MAIN window
      by sending the .. mof.cgi?viewcart string as the REDIRECT_URL.

      Example:
      name="POP_TARGET"
      value="MAIN"
      name="POP_AUTOCLOSE"
      value="0"
      name="REDIRECT_URL"
      value="http://www.gardenthemes.com/cgi-bin/mcart/mof.cgi?viewcart&previouspage=URL">

      Note: make sure and send the correct string including "&previouspage" exactly as defined, so MOFcart can have a valid "continue shopping" URL

    • POP_AUTOCLOSE value="15"

      NOTE: use the timer value or NULL "" (no auto close)
