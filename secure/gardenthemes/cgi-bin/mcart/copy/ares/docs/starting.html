Getting Started

Note: Read the section called Afer Installation in the Operating Documentation for more explanation about some of the adjustment to messages that are needed.

Going Live :

The Operating Documentation covers more about ARES operation. But you should test it all up enough to insure that ARES is working in all areas before you go live with it. When you're done testing, then you should empty out all the datafiles, and start with blank, empty files to go live with.

You should not attempt to manage the *.dat files manually, deleting individual records, because all 6 *.dat files have relational connections, and data is stored across all 6 files for a given Affiliate. The best way to preserve relational integrity among the data files is to let ARES manage everything. Once you are tested and ready to go live, simply zero out ALL the files and start over.

When you go live, your first two entries in the system should be yourself. If you zero out all the data files to purge all testing data, then you'll need to repeat Steps 1 through 4 in the Getting Started directions, so that you once again have Rep 100 & a Default Affiliate Registered.

Representative # 100:

Use your business name, real name, or code name and enter yourself as a Representative. You can only enter New Reps from the Admin Utility. Unless you actually use the first tier (Representative) level to pay earnings, then just enter a zero earning rate for yourself as Rep.

The "signup.html" form has a hidden field in it which submits Rep Number 100 as the Rep for all New Affiliate Sign Ups. If you disable this hidden input, and set Require Rep ON in the "signup.cgi" script, then ARES will prompt all new Affiliate Sign Ups to associate themselves with a valid Representative, presenting them a list of all registered Reps in ARES.

The Default Affiliate Number :

This will give your Admin Utility the ability to run stats on All orders placed from your site. If you do not specify a valid ARES default number, then MOF will only log transactions where it finds a valid Affiliate Number in the cookie, and you will lose opportunity to run stats on all orders.

You can set yourself at a zero earning rate since it is only for tracking purposes. Make sure you set the Customer Discount to zero for You (as Affiliate) unless you really plan to give customers without a valid Affiliate Number a discount. The default discount will apply to all customer who do not have a valid Affiliate Number in the cookie, and they will get the discount you specify in your Affiliate Record. Set to zero no customer discount will apply.
