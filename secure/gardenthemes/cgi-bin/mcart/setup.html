This is an example of what the <setup.cgi> install utility produced on first run, on an ideally configured linux/Apache Server. I'm installing to run the Front and Back Ends of the cart via Shared SSL, that is mapped nicely to the same Filespace as the regular Web Site filespace. Also, the server is configured to run scripts (SSL & nonSSL) CGI under the UID (user id) for the Virtual Space. That is nice cause it sets up the best security with only default permissions.

The installation site is here:
http://www.searchwise.org/mofcart/example_mofstore.html

So, in an ideally configured server, the only thing that I need to change is the 4 HTTPS paths below highlighted. The table directly below this example shows the 4 HTTPS (web) paths changed to what they should be. In this situation, everything is installed in about 2 minutes, with all examples, etc. set perfectly for the web site installed on.

About 90% of the Linux/Apache commercial servers I tested the install utility on did not have an Ideal set of configurations, but required some tweaking to fit the servers configs. Examples are: SSL was sloppy, mapped to seperate filespace & directories, SSL did not run under the owners UID, CGI scripts in general did not run under the owner's credentials, Hard drive paths (absolute paths) on the server were mapped in sloppy ways, virtual mapping for SSL filespace were awkward or not followed well from the CGI Environment, etc.

So, here's the bottom line on the install utility. You need to know your server, cause the utility can't really guess at all the variations out there.
