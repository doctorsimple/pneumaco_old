The Installation Utility : Read This

I spent about a month on the install utility. In the end, I realized that it was pretty grandiouse to think I could write an install utility that would work across the board on all servers. After testing the install utility on a handful of servers, I realized there was way too much variation on Commercial Hosting Servers, especially when the SSL configurations came into play.

Where I left things with the install utility is still pretty good, but it does require human oversight. You'll need to eyeball the settings. For now I've placed a file along side this one called <setup.html>. It give a very rough look at some issues that you need to watch when using the installation utility .. To be successful with the installation utility, you're going to have to know your server, and how SSL, paths, etc. are configured on it, if there are any pecularities, etc.

How To Install the ver2.5 package using the CGI install utility

  1. Create a Directory in your web site's ../cgi-bin/ called "mcart"
    If you already have a directory called "mcart" then you'll need to back it up and delete it. The install utility will stop if it detects anything in that DIR except the "unpack" material
  2. Upload the entire directory called "unpack" into the newly made "mcart" directory on your server
  3. Upload the <setup.cgi> script to the "mcart" directory on your server
  4. Set run permissions on the <setup.cgi> 755
  5. Make sure that the following Directories do not exist in your Web Root Space
    • ../ares
    • ../mofcart
    • ../invoices

      If those directories exist, back them up, and delete them from the server. The install utility needs to create it all. If it finds existing files, then it will stop and not overwrite anything.

  6. Run the <setup.cgi> script from your Web Browser
    --> http://www.site.com/cgi-bin/mcart/setup.cgi

Okay, that's it. If you got the settings correct, then everything started working like a charm. setup.cgi checks for permissions, and assigns permissions, passwords, etc. to everything, that's if the server is configured well enough to let it do that much. For example, Yahoo commercial space, doesn't allow CGI scripts to write .htacces files, etc.

There's lots of stuff that the server could screw up or inhibit <setup.cgi> from doing.

Setup.cgi Settings

There are two settings at the top of <setup.cgi>

Commented out around line 25 is a manual overwrite switch
# $overwrite_files = 0;

If you set it to positive, then <setup.cgi> will overwrite any files in its way
It will only overwrite files in the Directory Structure it installs to ..
Which are the directories in Steps 1 and 5 above
The switch is only there cause it was the only work around for some stubborn server pecularities
I would not use that switch unless you absolutely need to

There is a setting to manually select which of the two (2) example layouts you want to install

  1. "boxed" is the old layout, with very basic CSS
  2. "ver25" is pretty complex layout, with some serious CSS

$usepkg = 'boxed';
$usepkg = 'ver25';
